# Backlog Scrum

Este ficheiro apresenta a organização inicial das tarefas do projeto Sistema de Gestão de Restaurante, utilizando a metodologia Scrum.

## Product Backlog

O Product Backlog contém as funcionalidades principais previstas para o sistema.

- Criar login de utilizadores
- Criar gestão de reservas
- Criar gestão de mesas
- Criar gestão de pedidos
- Criar estados dos pedidos
- Criar gestão do menu
- Criar gestão de colaboradores
- Criar testes TDD para validação de funcionalidades

## Sprint Backlog inicial

O Sprint Backlog inicial contém as primeiras tarefas escolhidas pelo grupo para organizar o projeto.

- Criar repositório no GitHub
- Atualizar README do projeto
- Criar pasta src
- Criar pasta test
- Criar ficheiro de backlog Scrum
- Definir branches para funcionalidades
- Criar primeiros commits do grupo

## Estados das tarefas

As tarefas do projeto poderão passar pelos seguintes estados:

- Por fazer
- Em desenvolvimento
- Em revisão
- Concluído

## Organização do grupo

Cada elemento do grupo ficará responsável por pequenas tarefas do projeto. As alterações serão registadas no GitHub através de commits com mensagens claras.

Sempre que possível, as funcionalidades serão desenvolvidas em branches próprias e depois integradas na branch principal main.
